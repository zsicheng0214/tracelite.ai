研究与论文

六个组件维度，六组 bench-case

每个维度都对应一组可复现的工程任务与量化指标。我们在固定模型下做 ablation，把工程收益从模型能力里剥离出来。所有 bench-case 开源于 GitHub。

C1

上下文管理 / 记忆

Context & Memory
在有限的上下文窗口里，保留对当前任务最关键的信息。
优先上下文压缩 / 摘要
长轨迹下不同压缩策略对任务成功率与信息留存的影响。
指标：pass-rate · info-retention · token-budget
记忆调取
在何时、用何种检索把历史记忆带回当前推理。
指标：recall@k · relevance
渐进式披露 / 跨会话持久化
按需展开信息，并在多会话间稳定地维持状态。
在 GitHub 查看 bench-case
C2

工具层

Tool Layer
让模型在正确的时机，用正确的参数调用正确的工具。
优先工具描述 / schema 设计
schema 的措辞、示例与约束如何影响调用正确率。
指标：call-accuracy · arg-validity
工具筛选 / 输出处理
在大量工具中筛选候选，并稳健地解析工具返回。
指标：selection-precision
参数纠错
调用失败后，模型能否从报错中自我修复参数。
在 GitHub 查看 bench-case
C3

子 Agent / 编排

Orchestration
把任务在多个角色间切分，并把结果稳健地收敛回来。
优先何时停止 / 何时 spawn
主控 Agent 何时拆分子任务、何时收手，避免过度编排。
指标：over-spawn-rate · completion
协调拓扑
串行、并行还是层级，不同拓扑下的效率与正确性。
结果聚合
把子 Agent 的产出无损地合并为一致的最终结果。
在 GitHub 查看 bench-case
C4

错误恢复 / 鲁棒

Robustness
出错时能止损、归因，并把轨迹重新拉回正轨。
优先轨迹调节 / 重规划
检测到偏离后，能否重新规划而非一路错到底。
指标：recovery-rate · replan-cost
重试策略 / 降级回退
失败后的重试与回退如何兼顾成功率与成本。
循环检测
识别并打破无进展的重复循环。
在 GitHub 查看 bench-case
C5

验证 / Sensors

Validation
在定稿前，让 Agent 知道自己到底做对了没有。
优先验证门
在关键节点设置可量化的验证门，拦住错误产出。
指标：false-pass · false-block
自我批评 / 外部验证器
自检与外部验证器（编译、测试、规则）的组合效果。
完成度对照
逐条核对需求，避免"看起来完成"。
在 GitHub 查看 bench-case
C6

自进化

Self-evolution
从经验中安全地长出新能力，而不是无序膨胀。
优先受控增长 / 过滤
新技能在沉淀前的筛选与质量控制，防止污染。
指标：skill-quality · regression
技能合成 / 复用
把成功轨迹抽象成可复用技能，并在新任务中调用。
从失败中学习
把失败案例转化为可避免再犯的约束与经验。

Agent 试炼场

让你的 Agent，先在试炼场里跑一遍

一个面向 Agent 的测评与诊断空间。把你正在用的 Agent 接进来，在统一的 bench-case 上运行，并对每一条执行轨迹做细粒度归因。

测试体验环境

Agent 测试体验环境

一个隔离的 Docker 测评环境，预置 HarnessBench 的全部 bench-case。接入你的 Agent，固定模型、只变 harness，直接看到工程差异带来的指标变化。

  • 一条命令拉起隔离容器，支持任意 Agent / Harness 接入
  • 按维度选择 bench-case 套件，复现稳定、结果可比
  • 实时输出通过率、步数、工具调用、恢复率等 harness 指标
  • 导出结构化报告，方便横向对比不同 harness 设计

轨迹分析

Agent 执行轨迹分析

把一次运行的完整轨迹拆解成可读的步骤序列，定位 Agent 在哪一步偏离、为什么失败、哪一个 harness 组件是瓶颈 —— 让分数变得可归因。

  • step-by-step 可视化每一次推理、工具调用与状态变化
  • 自动标注失败点、循环、无效重试与恢复时刻
  • 按六个组件维度归因，告诉你瓶颈出在哪个环节
  • 支持多次运行对比，量化一次 harness 改动的真实收益
